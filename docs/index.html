Project Overview

一个围绕阅读、互动和 AI 辅助能力构建的多端系统

项目包含 Spring Boot 后端、桌面端 Web、移动端 H5，以及独立的轻量 TTS 服务。 功能重点放在图书阅读、书架管理、评论互动、好友社交、AI 问答、推荐和听书。

Java 17 / Spring Boot 3 Vue 3 / Vite MySQL / Redis / Elasticsearch Dify / TTS / WebSocket

功能范围

按业务拆分，当前系统的核心能力主要集中在阅读、社交、AI 和管理四个方向。

阅读与书架

  • 图书详情、章节阅读、阅读进度记录
  • 个人书架管理、继续阅读
  • 书单创建、查看、分享、导入

评论与社交

  • 图书评论、回复、点赞
  • 段落评论、段落分享
  • 好友申请、私聊、图书分享

AI 与听书

  • AI 阅读助手
  • 首页推荐
  • 段落朗读、整章听书、音频分享

系统结构

整体是典型的前后端分离结构，语音服务单独部署，方便替换和扩展。

前端层

  • reading-ui：桌面端页面
  • reading-mobile-ui：移动端页面
  • 通过 HTTP 接口与 WebSocket 与后端通信

服务层

  • src/main/java：核心后端业务
  • lightweight-tts-service：轻量听书服务
  • Dify、DashScope 等作为外部 AI 能力接入

主要模块

下面列的是项目里比较核心的业务模块，便于快速理解代码分布。

模块 说明
SysBookController 图书查询、上传、审核、推荐、排行榜
UserBookshelfController 书架增删、阅读进度同步、书架可见性切换
AuthController 验证码发送、注册、验证码登录、密码重置
DifyAiController AI 对话与流式响应转发
AiController 听书与音频生成入口
FriendController / ChatController 好友关系、聊天记录、未读消息管理
BookShareController / ParagraphShareController 图书分享与段落分享
BooklistController 书单创建、分享、导入

启动方式

按依赖服务、TTS 服务、后端、前端的顺序启动会比较稳定。

1. 准备基础依赖 启动 MySQL、Redis 和 Elasticsearch，并确认后端配置文件中的连接地址可用。
2. 启动轻量 TTS 服务 Docker 方式：docker compose -f docker-compose.tts.yml up -d --build
本地方式：进入 lightweight-tts-service 后运行 uvicorn app:app --host 0.0.0.0 --port 8091
3. 启动后端 在项目根目录运行 mvn spring-boot:run，或直接在 IDEA 中启动 ReadingSystemApplication。
4. 启动前端 桌面端：进入 reading-ui 执行 npm install、npm run dev
移动端：进入 reading-mobile-ui 执行 npm install、npm run dev
