智能阅读系统

前后端分离的在线阅读平台，覆盖图书阅读、书架管理、评论互动、好友私聊、AI 问答、知识库同步、推荐和听书。

支持 Docker Compose 一键部署，含 Flyway 数据库迁移、基础设施降级容错和多端前端。

Java 17 / Spring Boot 3.2 Vue 3 / Vite / Element Plus / Vant MySQL 8 / Redis 7 / Elasticsearch 8 Dify / edge-tts Docker Compose / Nginx Flyway / MyBatis-Plus / WebSocket
GitHub 架构文档 API 参考

功能范围

按业务方向组织，覆盖阅读、社交、AI 和管理四大领域。

阅读与书架

  • 图书详情、目录解析、章节阅读
  • 阅读进度自动保存与恢复
  • 个人书架管理、继续阅读
  • 书单创建、分享码导入
  • 自定义阅读主题（护眼/暗夜/适老）

评论与社交

  • 书评、回复、点赞
  • 段落评论、段落分享
  • 好友搜索、申请、列表
  • 私聊、已读未读、消息通知
  • 图书/段落/音频分享给好友

AI 与听书

  • Dify AI 阅读助手 SSE 流式对话
  • AI 个性化首页推荐 + 熔断降级
  • 知识库章节级自动同步
  • 段落朗读 + 整章听书
  • 元数据过滤知识库检索

工程实践

项目在快速迭代中积累的工程化能力。

基础设施与测试

  • Docker Compose 一键部署（MySQL + Redis + ES + 后端 + 双前端 + TTS）
  • 多阶段 Docker 构建（Maven → JRE，Node → Nginx）
  • 降级容错：无 Redis/ES 时自动退到本地内存缓存
  • JUnit 5 + Mockito 后端测试，Vitest 前端组件测试
  • GitHub Actions CI：后端编译测试 + 双前端构建

架构与代码质量

  • Flyway 数据库迁移基线化
  • MyBatis-Plus 分页 + Elasticsearch 搜索（含 MySQL 降级）
  • 推荐熔断器：Dify 异常时 5 分钟短路 + 随机回退
  • Read.vue 组件拆分：6 composable + 9 组件的模块化架构
  • CSS 设计系统：自定义属性 + 毛玻璃 + 暗色模式 + 无限滚动
  • XSS 防护：Markdown 渲染经 DOMPurify 净化
技术亮点 — 后端采用 @ConditionalOnProperty + @Autowired(required=false) + ConcurrentHashMap 三级降级策略，保证无 Redis/ES 环境仍可完整运行； 前端 Read.vue 从 1700 行拆分为 6 个 composable + 9 个子组件，桌面端/移动端共享结构； DegradeEnvironmentPostProcessor 在 Spring Boot 启动阶段按开关排除自动配置，避免 Bean 缺失异常。

启动方式

推荐 Docker Compose 一键部署，30 秒内启动全部服务。

Docker Compose（推荐） cp .env.example .env → 编辑填入 MYSQL_PASSWORD → docker compose up -d
启动后访问 桌面端 :5173 / 移动端 :5174 / 后端 :8090。
手动启动 确保 MySQL 8 + Redis + Elasticsearch 可用 → Flyway 自动建表 → mvn spring-boot:run 启动后端 → 进入 reading-ui 和 reading-mobile-ui 分别 npm run dev。
可选 AI 能力 TTS 听书需要启动 lightweight-tts-service；AI 助手和推荐需要配置 Dify Key。
